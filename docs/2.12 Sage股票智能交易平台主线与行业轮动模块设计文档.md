# Sage股票智能交易平台主线与行业轮动模块设计文档

## 1. 模块定位
识别市场“主线板块”和行业轮动节奏，为选股排序与信号融合提供**行业权重与趋势加成**。

---

## 2. 输入 / 输出
**输入**：
- 行业指数（申万行业指数）
- 概念/板块指数（同花顺 ths_index / ths_daily）
- 行业/概念成分股
- 资金/成交额/涨停强度（可选）

**输出**：
```json
{
  "date": "2026-02-13",
  "sector_scores": [
    {"industry": "新能源车", "score": 86, "label": "MAINLINE"}
  ]
}
```

**输出落地（建议）**：
- `data/signals/mainline_score_<YYYYMMDD>.parquet`
- 字段：`trade_date/industry/score/label`
- 概念热度：`data/processed/concepts/concept_heat.parquet`
- 概念信号：`data/signals/concept_signals.parquet`

**当前实现落地**：
- `data/signals/industry/industry_score_snapshot_latest.parquet`（行业综合分）
- `data/signals/industry/industry_signal_snapshot_latest.parquet`（policy/concept/northbound 对齐快照）
- 周流程执行层通过 `config/app/strategy_governance.yaml -> industry_signals.overlay` 将主线综合分与行业信号共同作用于 `execution_signals_<date>.parquet`

---

## 3. 核心指标
1. **领涨强度**（相对指数收益）
2. **扩散速度**（上涨家数占比）
3. **持续性**（持续周数）
4. **兑现度**（财报或盈利支撑）
5. **拥挤度**（高换手/涨停占比，负向）

### 3.1 计算示意（简化）
```
leadership = Ret_industry_4w - Ret_index_4w
diffusion  = 上涨股票占比
persistence = 连续强势周数
crowding = 换手率/涨停比例
```

## 4. 概念信号（补充）
**定位**：概念不是行业，不做 NBS→申万映射。概念仅作为“行业主线/轮动”的辅助信号（偏置或风险提示）。

**输入**：
- 同花顺概念指数（`ths_index` / `ths_daily`）
- 概念成分股（可选，用于行业覆盖判断）

**输出**：
- `data/signals/concept_signals.parquet`（全量）
- `data/signals/concept_signals_top10.parquet`（仅热点Top10）
  - 字段：`concept_code/concept_name/trade_date/heat_20d/heat_60d/concept_heat_score/overheat_flag`

字段示例：
- `trade_date`
- `concept_code`
- `concept_name`
- `heat_20d/heat_60d`
- `momentum_20d/momentum_60d`
- `overheat_flag`

**使用规则**：
- 不直接触发交易，仅作为**行业评分偏置**或**风险提示**
- 若某概念覆盖某行业成分股比例高 → 行业权重加成
- 若概念热度过高 → 行业权重降温或风险提示
 - 概念→行业覆盖率表：`data/processed/concepts/concept_industry_coverage.parquet`

## 7. 未来规划（主线模型V2：豆包方案落地）
### 7.1 V2定位
- 采用“**先门控、再评分**”框架：先判定是否具备牛市土壤，再在行业横截面做四维主线评分。
- 四维评分：`产业边际 / 资金共识 / 业绩兑现 / 结构有效性`。
- 输出目标：`Top2主线行业 + 行业分数 + 维度贡献 + 置信度`，用于后续选股与仓位联动。

### 7.2 数据源分层（避免卡在高门槛数据）
- **L0（必备免费）**：Tushare行业指数/北向/成交、政府网站政策、东方财富行业研报、同花顺板块指数。
- **L1（可选增强）**：Wind/iFind/Choice 一致预期与行业细分景气库。
- **降级规则**：当L1不可用时，自动退回L0代理特征，保持模型可运行。

### 7.3 V2模块拆分
- `bull_regime_gate`：牛市土壤门控（趋势结构+市场广度+流动性+盈利预期代理）。
- `mainline_v2_scorer`：四维评分主引擎（支持静态权重与IC/IR动态权重）。
- `mainline_v2_features`：四维特征计算层（含代理特征与缺失处理）。
- `mainline_v2_output`：标准化信号输出与诊断报表（维度贡献、排名变化、覆盖率）。

### 7.4 里程碑（建议）
- **M1（1周）**：完成门控与四维最小特征集（L0数据），输出可回测主线分数。
- **M2（1~2周）**：接入动态权重（IC/IR滚动）与A/B回测（V1 vs V2）。
- **M3（1周）**：接入风控联动（行业上限、拥挤度降温、趋势降仓）与监控告警。

### 7.5 风险与对策
- 一致预期/机构持仓数据可得性不足 → 使用业绩上修代理特征并记录置信度。
- 行业映射口径漂移 → 固化申万L1口径，输出映射审计报告。
- 维度过多导致过拟合 → 先固定小特征集，周频回测后再逐步扩展。
